Section 1 — What Is This Tool?

The big picture in plain English

The Solrei CPT Dashboard is a private, internal web tool that answers one critical question:

💡
"Which insurance companies are underpaying us, by how much, and what should we ask for?"
It compares your contracted payer rates against Medicare 2026 benchmark rates across all 23 states where Solrei has licensed providers — so you always know which contracts to prioritize in negotiations.

No patient data is involved. No data leaves your network. This is your private database running on your own hardware.

Section 3 — Starting & Stopping the Tool

How to control the server on the Mac Mini

ℹ️
Think of uvicorn as the power switch for the dashboard. When it's running, the dashboard is live. When it's stopped, no one can access it. It runs on the Mac Mini.
🟢 Check If the Server Is Running

From your iMac Terminal, run this first to see if uvicorn is already active:

Run from iMac Terminal (via SSH) ssh sbhserver1@192.168.88.178 "ps aux | grep uvicorn"

If you see a line containing backend.main:app and port 8000, the server is running. If the only result is the grep command itself, it's not running.

🟢 Start the Server

Run this from your iMac Terminal to start the server on the Mac Mini and keep it running even after you close Terminal:

Run from iMac Terminal ssh sbhserver1@192.168.88.178 "cd /Users/sbhserver1/cpt_dashboard && nohup /opt/homebrew/Cellar/python@3.11/3.11.15_1/Frameworks/Python.framework/Versions/3.11/Resources/Python.app/Contents/MacOS/Python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000 --app-dir /Users/sbhserver1/cpt_dashboard > /Users/sbhserver1/cpt_dashboard/uvicorn.log 2>&1 &"
✅
After starting, wait 5 seconds then visit http://192.168.88.178:8000/dashboard in your browser to confirm it's running.
🔴 Stop the Server
Run from iMac Terminal ssh sbhserver1@192.168.88.178 "pkill -f 'uvicorn backend.main'"
🔄 Restart (after deploying updates)

Stop first, then start:

Step 1 — Stop ssh sbhserver1@192.168.88.178 "pkill -f 'uvicorn backend.main'"
Step 2 — Start ssh sbhserver1@192.168.88.178 "cd /Users/sbhserver1/cpt_dashboard && nohup /opt/homebrew/Cellar/python@3.11/3.11.15_1/Frameworks/Python.framework/Versions/3.11/Resources/Python.app/Contents/MacOS/Python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000 --app-dir /Users/sbhserver1/cpt_dashboard > /Users/sbhserver1/cpt_dashboard/uvicorn.log 2>&1 &"
🚒 "Address Already In Use" Error

If restart fails with this error, force-clear port 8000 first:

Force-kill anything on port 8000 ssh sbhserver1@192.168.88.178 "lsof -ti:8000 | xargs kill -9"

Then run the start command above.

📋 Check Server Log

To see what the server has been doing (errors, requests, etc.):

View last 50 log lines ssh sbhserver1@192.168.88.178 "tail -50 /Users/sbhserver1/cpt_dashboard/uvicorn.log"

Section 4 — Deploying Updates to the Server NEW

Step-by-step: how to get changes from your iMac to the live Mac Mini server

⚠️
Always deploy in the correct order. Files first, database changes second, server restart last. Skipping steps or doing them out of order can break the dashboard.

Figure 3 — The 4-step deployment process from iMac → Mac Mini server

📋 Step-by-Step: Typical Full Deploy

Here are the exact commands for a typical deploy session when Claude has made changes to the dashboard. Run each one from your iMac Terminal:

  1. 1

    Copy dashboard.html (if changed)

    This is the most common file to update — any visual or UI change lives here.

    scp "/Users/deanpedersen/Projects/solrei/PROJECT - ACTIVE/Hosted on sbhserver1/CPT_Dashboard/dashboard.html" sbhserver1@192.168.88.178:/Users/sbhserver1/cpt_dashboard/dashboard.html
  2. 2

    Copy any other changed files

    Repeat the scp pattern for any other file that was edited (Python files, SQL scripts, CSV files). Just change the filename at the end of both paths.

    # Template — replace FILENAME with the actual file name scp "/Users/deanpedersen/Projects/solrei/PROJECT - ACTIVE/Hosted on sbhserver1/CPT_Dashboard/FILENAME" \ sbhserver1@192.168.88.178:/Users/sbhserver1/cpt_dashboard/FILENAME
  3. 3

    Run SQL scripts (if any)

    If a new SQL file was created (numbered like 24_something.sql), run it on the Mac Mini's database:

    ssh sbhserver1@192.168.88.178 \ "cd /Users/sbhserver1/cpt_dashboard && \ /opt/homebrew/opt/postgresql@16/bin/psql solrei_cpt -f sql/filename.sql"
  4. 4

    Import CSV rates (if any)

    If new rate CSVs were generated, import them directly via the API — no need to SSH for this step:

    curl -X POST "http://192.168.88.178:8000/api/intermediaries/import" \ -F "file=@/Users/deanpedersen/Projects/solrei/PROJECT\ -\ ACTIVE/Hosted\ on\ sbhserver1/CPT_Dashboard/data/filename.csv"
  5. 5

    Restart the server

    Required whenever dashboard.html or any Python .py file was changed. Not needed if only the database was updated.

    # Stop ssh sbhserver1@192.168.88.178 "pkill -f 'uvicorn backend.main'" # Wait 3 seconds, then start ssh sbhserver1@192.168.88.178 "cd /Users/sbhserver1/cpt_dashboard && nohup /opt/homebrew/Cellar/python@3.11/3.11.15_1/Frameworks/Python.framework/Versions/3.11/Resources/Python.app/Contents/MacOS/Python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000 --app-dir /Users/sbhserver1/cpt_dashboard > /Users/sbhserver1/cpt_dashboard/uvicorn.log 2>&1 &"
  6. 6

    Verify in browser

    Open http://192.168.88.178:8000/dashboard and confirm your changes are visible. Hard refresh with Cmd + Shift + R if needed.

💡
Quick Tip: What Files Get Changed Most Often?
dashboard.html — any visual/UI change · SQL files — new payers, updated database structure · CSV files — new rates from Google Sheets · backend/*.py — backend logic changes (rare)

Section 7 — Database Management

Working with the PostgreSQL database (solrei_cpt) on the Mac Mini

ℹ️
The database is named solrei_cpt and runs on PostgreSQL 16 on the Mac Mini. Think of it as a super-organized filing cabinet that stores every rate, payer, and contract in the tool. The psql program is how you talk to it.
Open the Database Console
From iMac Terminal (via SSH) ssh sbhserver1@192.168.88.178 "/opt/homebrew/opt/postgresql@16/bin/psql solrei_cpt"

This opens an interactive session. Type \q to exit.

Run a SQL Script File
From iMac Terminal (replace filename.sql) ssh sbhserver1@192.168.88.178 \ "cd /Users/sbhserver1/cpt_dashboard && \ /opt/homebrew/opt/postgresql@16/bin/psql solrei_cpt -f sql/filename.sql"
Useful Quick Queries
What You Want to SeeCommand (run via SSH or direct psql)
All payers in the database psql solrei_cpt -c "SELECT payer_name FROM payers ORDER BY payer_name;"
Medicare rates loaded for all 23 states psql solrei_cpt -c "SELECT state, COUNT(*) FROM benchmark_fee_schedule WHERE source_name = 'Medicare 2026' GROUP BY state ORDER BY state;"
Most recent rate upload per intermediary psql solrei_cpt -c "SELECT intermediary_name, state, MAX(updated_at) FROM intermediary_rates GROUP BY intermediary_name, state ORDER BY intermediary_name, state;"
SQL Migration Files

Files numbered 01–23+ in the sql/ folder built and updated the database schema over time. Do not re-run them unless rebuilding from scratch — they are a historical record, not a maintenance task.

Section 10 — Troubleshooting

Common problems and exactly how to fix them

⚠️ Dashboard won't load / "Could not connect to API"
The server (uvicorn) is not running. Start it using the command in Section 3. Then wait 5 seconds and refresh the browser.
ssh sbhserver1@192.168.88.178 "ps aux | grep uvicorn" # Check if running
⚠️ "Address already in use" when starting server
Port 8000 is still occupied by a previous process. Force-clear it:
ssh sbhserver1@192.168.88.178 "lsof -ti:8000 | xargs kill -9"
Then run the start command again (Section 3).
⚠️ I made changes but the dashboard looks the same
Two possible causes:

1) Files not deployed to Mac Mini yet — Follow Section 4 to copy files via scp and restart the server.

2) Browser cache — Your browser remembered the old version. Press Cmd + Shift + R to force a fresh load. If still stuck, try an Incognito window.
⚠️ Payer doesn't appear in the dashboard
Some payers are state-specific and only appear when the correct state is selected. For example, Regence BlueCross BlueShield of Oregon only appears when OR is selected in the state dropdown. Check the state selector in the dashboard header first.
⚠️ Dashboard loads but shows no rate data
The database may not have rates imported, or the database itself may be down. Verify:
ssh sbhserver1@192.168.88.178 "/opt/homebrew/opt/postgresql@16/bin/psql solrei_cpt -c 'SELECT COUNT(*) FROM fee_schedule_lines;'"
If you get a number greater than 0, the database has data. If you get an error, PostgreSQL may need to be started on the Mac Mini.
⚠️ MASTER_DATA tab shows #ERROR in Google Sheets
The IMPORTRANGE formula is broken. Go to the MASTER_DATA tab, click cell A1, and type (do not paste) this formula exactly — using straight quotes, not curly quotes:
=IMPORTRANGE("1okxQgWz400amUqC_i-4C8AJcFvZkjD0f2yjl87MIT-c","Rates!A1:AX200")
When prompted, click Allow Access.
⚠️ Amber ⚠ warning in the UPDATED column
Those rates are more than 90 days old and may be stale. Run the quarterly Google Sheets sync (Section 5.3) to refresh and re-upload the CSV.
⚠️ scp says "Permission denied" or "No such file"
Check that the file path on your iMac is correct and the folder name is quoted correctly (spaces in the path need quotes). Also verify the destination path exists on the Mac Mini:
ssh sbhserver1@192.168.88.178 "ls /Users/sbhserver1/cpt_dashboard/"
⚠️ git push fails with 403 error
GitHub no longer accepts account passwords for git operations. You need a Personal Access Token (PAT). Go to github.com → Settings → Developer Settings → Personal Access Tokens → Generate new token. Use that token as your password when git asks. Ask Claude if you need help.
⚠️ Apps Script times out during quarterly sync
Use populate_slave_v8.gs — this is the values-based version that reads MASTER directly and completes in about 10 seconds. It does not use slow formula chains.
